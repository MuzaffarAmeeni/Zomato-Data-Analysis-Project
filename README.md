📊 Zomato Operations & Performance Analytics
A Comprehensive Data Modeling & BI Project

📌 Project Overview
This project focuses on transforming raw, unstructured restaurant data from Zomato into a multi-dimensional data model to drive business insights. The goal was to analyze restaurant performance, customer preferences, and regional trends to help stakeholders make data-driven decisions regarding expansion and service optimization.

🛠️ Tech Stack & Skills
Data Tool: Microsoft Power BI / Excel

Language: DAX (Data Analysis Expressions) for complex measures

Data Modeling: Star Schema (Fact and Dimension tables)

ETL Process: Power Query for data cleaning and transformation

Key Techniques: Calendar Table creation, KPI Card design, and Interactive Slicers

🚀 Key Features & Analysis
Data Modeling: Constructed a robust Star Schema by creating dedicated Dimension tables for Geography, Cuisines, and Date (Calendar Table), linked to a central Fact table.

KPI Development: Built dynamic metrics to track:

Average Rating vs. Cost for Two

Percentage of Restaurants with Online Delivery

City-wise Distribution of Cuisines

Geospatial Insights: Utilized map visuals to identify high-density restaurant hubs and underserved areas.

Interactive UX: Integrated drill-down features and custom slicers for a seamless user experience.

💡 Business Insights (The "So What?")
Online vs. Offline: Identified that restaurants offering "Online Delivery" maintain a 15% higher average rating compared to those without.

Price Sensitivity: Discovered a "sweet spot" price range for the Visakhapatnam region that maximizes both volume of orders and customer satisfaction.

Cuisine Trends: North Indian and Chinese cuisines dominate the market, but there is a significant gap in niche "Healthy Food" categories in high-income neighborhoods.
